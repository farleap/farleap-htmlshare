# ADR-0004: インライン（範囲アンカー）コメント方式

- Status: Accepted
- Date: 2026-06-28

## Context

Phase 2 でレビュー（コメント）機能を導入する。目的は、AI 生成 HTML に対して社内のレビュアーが
**文書の該当箇所を指しながら**フィードバックを残せること。「3段落目のこの数値の根拠は？」を、
チャットの曖昧な参照ではなく、本文に固定された指摘として残したい。

制約として、プレビューは不透明オリジンの sandbox iframe（[ADR-0001](./0001-opaque-origin-iframe-isolation.md)）であり、
親（App）から iframe 内のテキスト選択や DOM へ**直接は触れない**。
インライン方式を採るには、この隔離を壊さずに「選択範囲を親へ受け渡す」経路が必要になる。

## Considered Options

1. **文書全体への単一スレッド**（アンカー無し。資料1件 = 1コメント欄）
2. **インライン（範囲アンカー）**（本文の特定範囲にコメントを固定。Google Docs 風）

## Decision

**選択肢 2 を採用。** 本文のテキスト範囲を選択してコメントを紐づける。
アンカーは **引用テキスト＋前後文脈＋位置**（`comments` テーブルの anchor 列群。W3C Web Annotation の
TextQuoteSelector / TextPositionSelector に相当）で表現する。

隔離を壊さない実現方法：

- Content 面が配信する HTML に、**レビューモード時だけ**表示用スクリプトを注入する。
  役割は「選択を検知してアンカーを生成し親へ `postMessage`」と「既存コメントのハイライト描画」。
- iframe は**不透明オリジンのまま**（`allow-same-origin` は付けない）。注入スクリプトも親 Cookie/DOM には到達不可。
- 親 ⇄ iframe の `postMessage` は**双方がオリジンを厳密検証**する。
- 本文 DOM は書き換えず、ハイライトは非破壊で重ねる。

## Consequences

- **Positive:**
  - 「どこの話か」が本文に固定され、AI 生成物の具体的な指摘という主目的を満たす。
  - セキュリティ不変条件（[ADR-0001](./0001-opaque-origin-iframe-isolation.md)）を維持したまま実現できる。
  - 漸進的開示：通常閲覧では一切注入せず、表面積を増やさない。
- **Negative / トレードオフ:**
  - 注入スクリプト＋ postMessage という間接経路が必要で、文書全体スレッド方式より実装が重い。
  - テキストが書き換わるとアンカーがずれ得る問題が生じる
    → 版固定と再アンカーで対処（[ADR-0005](./0005-version-pinned-comments-and-reanchoring.md)）。

## なぜ文書全体スレッド（選択肢1）を退けたか

実装は最も軽い。だが「本文のどこを指すか」を表現できないため、レビューが「全体への感想」に留まり、
AI に戻して直させる具体性を持てない。主目的（箇所を指した指摘）を満たさないため、起点としては不適。
（軽さを評価し、Phase 2 の最小段階で「アンカー無しコメントから出す」折衷も検討したが、
版固定・再アンカーの土台を最初から通したほうが手戻りが無いと判断し、インラインで一気に通すことにした。）
